Case 01 · Provenance Analysis

Training Data Provenance Through Output Analysis

LLM Evaluation AI Output Analysis Training Data Digital Forensics Hallucination Detection
Read full case ↗

Investigated recurrent patterns in AI-generated subtitles for streaming platforms, treating output anomalies not as random errors but as traceable artifacts of training data composition. Developed a replicable 5-stage provenance audit framework combining corpus linguistics and digital forensics.

⚠ Before — AI output anomaly

Whisper-generated subtitle with zero acoustic or contextual alignment with source media. Platform-specific attribution patterns preserved verbatim in commercial streaming output — invisible to standard QA processes.

✓ After — Provenance identified

Artifact classified as volunteer-sourced training data trace. Pattern traced to specific platform, enabling targeted dataset audit and curation recommendations for future model training.

Three artifact types classified: volunteer-sourced data traces, cross-platform contamination, and unauthorized content inclusion — each requiring different detection methodology and curation response.

Case 02 · Corpus Linguistics at Scale

From Speech Analytics to Enterprise Taxonomy: Linguistic Variation at Scale

Corpus Linguistics NLP Data Quality Semantic Analysis Knowledge Systems Taxonomy Design
Read full case ↗

Analyzed 2M+ monthly customer-agent interactions at a major telecom to map systematic linguistic variation between how customers expressed problems and how systems classified them — the same type of noise that degrades AI model performance at scale.

⚠ Before — Triple mismatch

Customer: "my mobile data ran out"
Agent search: "no mobile internet"
System label: "data allowance exhaustion"

Three layers of variation. Retrieval failure for 30,000+ agents daily.

✓ After — Structured vocabulary

41 customer variants + 12 agent patterns normalized into governed taxonomy. Full semantic relation framework (BT/NT/RT/UF per ISO 25964). Deployed enterprise-wide.

Controlled vocabulary of 2,700+ terms built using Boolean proximity operators and corpus linguistics methods. Directly applicable to NLP dataset curation, AI output evaluation, and language model fine-tuning.

Python Workflow Automation for SDH Linguistic Metadata

Python Text Processing AI Workflow Metadata Structuring NLP Pipeline
Read full case ↗

Built a Python script for automated SDH metadata structuring — parsing subtitle files, identifying timestamps, and inserting standardized annotation fields. Architecture mirrors NLP annotation pipelines used in AI training data preparation.
